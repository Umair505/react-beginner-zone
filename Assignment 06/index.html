The difference between var, let, and const

var: Old way to declare variables. It can be changed and used anywhere in the code.

let: It can be changed, but only works where you declare it.

const: It cannot be changed after you declare it. Use it for things that stay the same. üòä

Difference between map(), forEach(), and filter()

map(): Returns a new array with transformed elements.

forEach(): Loops through elements, no return.

filter(): Returns a new array with elements that pass a condition.

Explain arrow functions and how they are different from regular functions

Arrow functions are a shorter and cleaner way to write functions. They don‚Äôt have their own this, meaning they inherit this from the surrounding scope ‚Äî this makes them perfect for callbacks and when we need to preserve context. Regular functions, however, do have their own this, which changes depending on how the function is called. Additionally, arrow functions can't be used as constructors, but regular functions can.

How JavaScript Promises work

A Promise is an object in JavaScript that handles asynchronous operations. It can be in one of three states: pending (still working), resolved/fulfilled (success), or rejected (error). When a Promise succeeds, it calls the resolve() function, and you handle it with then(). If something goes wrong, it calls reject(), and you handle errors with catch().

How closures work in JavaScript

A closure happens when a function "remembers" the variables from its outer scope, even after that outer function has finished running. For example, if we create a function inside another function, and the inner function uses a variable from the outer one, that variable stays alive in memory, even though the outer function is done.
